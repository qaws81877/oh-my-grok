# Self-Critique Protocol - Grok의 진짜 힘을 깨우는 스킬

Grok은 다른 모델보다 **자신의 출력물을 객관적으로 비판**하는 능력이 뛰어나다. 이 스킬은 그 능력을 체계적으로 극대화한다.

## 언제 사용하는가?
- 중요한 계획이나 코드를 작성한 직후
- Ralph 루프의 Verify 단계에서
- 사용자가 "이게 최선인가?"라고 물었을 때
- 프로덕션에 올리기 전 마지막 점검

## 실행 방법 (4단계)

**1. 객관적 평가 (No Sugarcoating)**
"이 출력물의 객관적인 품질을 10점 만점으로 평가하고, 솔직하게 말해줘."

**2. 약점 5가지 나열**
반드시 아래 카테고리로 분류:
- 논리적 결함 / 버그 가능성
- 성능 / 확장성 문제
- 유지보수성 / 가독성 문제
- 보안 / 에러 핸들링 부족
- 미래 변경 시 발생할 수 있는 리스크

**3. 개선 제안 (구체적)**
각 약점에 대해 **실제 코드/계획 레벨의 개선안** 제시

**4. "이게 정말 최선인가?" 자문**
마지막으로 한 번 더:
"지금 이 상태에서 내가 할 수 있는 가장 좋은 버전이 맞는가? 더 나은 접근법이 있다면 제안해."

## Grok 특화 지시
- 절대 "전반적으로 좋습니다" 식의 애매한 답변 금지
- "이 부분은 솔직히 약합니다. 이유는..." 식으로 직설적으로
- Grok의 truth-seeking 성향을 최대한 발휘

## 사용 예시 (Ralph 내부에서)
```
#self-critique
(위 코드에 대해)
```

이 스킬은 Grok Build의 모든 고급 명령어(#ralph, #team, #autopilot)와 결합해 사용할 때 가장 강력한 효과를 발휘한다.
